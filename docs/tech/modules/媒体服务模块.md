## 1. 模块职责

| 职责 | 说明 |
|------|------|
| 上传凭证 | 生成 COS 预签名 PUT URL |
| 路径规范 | 按 purpose 隔离目录 |
| 上传确认 | 记录 media_objects，防未授权 URL 入库 |
| 限制校验 | 类型、大小、扩展名 |
| CDN 公网 URL | 返回可写入业务表的 publicUrl |

**不负责**：视频转码（MVP 客户端直传 MP4）；图片裁剪（客户端处理）。

## 5. 功能详细设计

### 5.1 获取上传凭证

**POST `/media/upload-credential`**

**Request**

```json
{
  "mediaType": "video",
  "purpose": "record_video",
  "fileExt": "mp4",
  "fileSize": 18350080
}
```

**校验**

1. purpose 与 mediaType 匹配（见上表）
2. fileExt 合法
3. fileSize ≤ maxSize
4. 频率未超限

**Response**

```json
{
  "uploadUrl": "https://bucket.cos.ap-guangzhou.myqcloud.com/...",
  "objectKey": "prod/uuid/record/video/xxx.mp4",
  "publicUrl": "https://cdn.example.com/prod/uuid/record/video/xxx.mp4",
  "expireAt": "2026-06-18T10:15:00.000Z",
  "headers": {
    "Content-Type": "video/mp4"
  }
}
```

**服务端内部**

```typescript
async getUploadCredential(userId, req) {
  policyValidator.check(req);
  const objectKey = pathBuilder.build(userId, req.purpose, req.fileExt);
  const uploadUrl = cosClient.presignedPut(objectKey, req.fileSize, ttl=900);
  await mediaRepo.insertPending({ objectKey, userId, ... });
  return { uploadUrl, objectKey, publicUrl, expireAt, headers };
}
```

**客户端上传**

```javascript
// 伪代码：微信小程序
wx.uploadFile({
  url: credential.uploadUrl,
  filePath: localTempPath,
  name: 'file',
  header: credential.headers,
  success: () => confirmUpload(credential.objectKey)
});
```

> 若 COS 预签名 PUT 与 `wx.uploadFile` 不兼容，可改用 **POST Object** 策略表单上传（实现时在 cos.client 封装）。

### 5.2 确认上传

**POST `/media/confirm`**

```json
{ "objectKey": "prod/uuid/record/video/xxx.mp4" }
```

**逻辑**

1. 查 `media_objects` WHERE object_key AND user_id = auth.userId
2. 若不存在 → 40406
3. HEAD COS 对象，确认存在且 size > 0
4. UPDATE status = `confirmed`, confirmed_at = NOW()
5. 返回 MediaMeta

**Response**

```json
{
  "objectKey": "...",
  "publicUrl": "https://cdn...",
  "mediaType": "video",
  "purpose": "record_video",
  "fileSize": 18350080,
  "status": "confirmed"
}
```

**业务表写入时机**：客户端拿 `publicUrl` 填入赛道/成绩/帖子表单，随业务 POST 一并提交；业务 Service 可选校验 URL 域名与 objectKey 已在 media_objects confirmed。

### 5.3 业务模块 URL 校验（可选钩子）

```typescript
// shared/mediaGuard.ts — 供 M02/M03/M04 调用
async assertOwnedMediaUrl(userId: string, url: string, expectedPurpose: string) {
  const objectKey = cdnUrlToKey(url);
  const row = await mediaRepo.findConfirmed(objectKey);
  if (!row || row.userId !== userId || row.purpose !== expectedPurpose) {
    throw new ForbiddenError('媒体文件无效');
  }
}
```

MVP 可只做域名白名单；confirm 机制为推荐安全基线。

## 8. 客户端封装

```typescript
// services/media.ts
async function uploadMedia(localPath: string, options: UploadOptions): Promise<string> {
  const stat = await wx.getFileInfo({ filePath: localPath });
  const cred = await post('/media/upload-credential', {
    mediaType: options.mediaType,
    purpose: options.purpose,
    fileExt: options.ext,
    fileSize: stat.size
  });
  await putFile(cred.uploadUrl, localPath, cred.headers);
  const meta = await post('/media/confirm', { objectKey: cred.objectKey });
  return meta.publicUrl;
}
```

**进度 UI**：`wx.uploadFile` 的 `onProgressUpdate` 绑定页面进度条。

## 10. 安全

| 项 | 措施 |
|----|------|
| 鉴权 | credential 必须登录 |
| 路径 | objectKey 含 userId，禁止跨用户 confirm |
| 有效期 | 预签名 15 分钟 |
| 内容类型 | Content-Type 与 ext 一致 |
| 防盗链 | CDN Referer 白名单（小程序域名） |
