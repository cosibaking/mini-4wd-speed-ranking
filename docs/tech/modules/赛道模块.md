## 3. 功能详细设计

### 3.1 创建赛道（分步表单 → 一次提交）

产品上分 3 步，**技术实现为客户端缓存草稿，最终一步 POST**（减少中途失败）。

**POST `/tracks`**

```json
{
  "name": "朝阳公园北广场赛道",
  "location": {
    "lat": 39.9321000,
    "lng": 116.4547000,
    "address": "北京市朝阳区朝阳公园北路..."
  },
  "organizerName": "张三",
  "organizerContact": "wxid_xxx",
  "lengthMeters": 120,
  "floorPlanUrls": [
    "https://cdn.../fp1.jpg",
    "https://cdn.../fp2.jpg"
  ],
  "exampleVideoUrl": "https://cdn.../example.mp4",
  "ruleNote": "从起跑线到终点线，视频需连续无剪辑。"
}
```

**字段校验**

| 字段 | 规则 |
|------|------|
| name | 必填，1–20 字，`trim` |
| location.lat/lng | 必填，中国境内合理范围（lat 3–54, lng 73–136） |
| location.address | 必填，≤ 255 字 |
| organizerName | 必填，≤ 64 字 |
| organizerContact | 选填，≤ 128 字 |
| lengthMeters | 选填，1–10000 |
| floorPlanUrls | 选填，0–3 个 HTTPS URL |
| exampleVideoUrl | 选填，须为已 confirm 的 media 或合法 CDN URL |
| ruleNote | 选填，≤ 500 字 |

**业务规则**

1. `isNameDuplicate(creatorId, name)` → 40900
2. 写入 `tracks` + 批量写入 `track_floor_plans`
3. `creator_id = auth.userId`
4. 返回完整 `TrackDetail` + `leaderboardSummary`（初始为空）

**客户端分步映射**

| Step | 字段 |
|------|------|
| 1 | name, location, organizerName, organizerContact |
| 2 | lengthMeters, floorPlanUrls |
| 3 | exampleVideoUrl, ruleNote |

### 3.2 编辑赛道

**PATCH `/tracks/:id`**

- 权限：`track.creatorId === auth.userId`，否则 40300
- 可部分更新；`name` 变更仍做同主理人下唯一校验
- 平面图：全量替换策略（传 `floorPlanUrls` 数组则 delete+insert）

### 3.3 赛道详情

**GET `/tracks/:id`**

**Response `data`**

```json
{
  "id": "uuid",
  "name": "朝阳公园北广场赛道",
  "location": { "lat": 39.93, "lng": 116.45, "address": "..." },
  "organizerName": "张三",
  "organizerContact": "wxid_xxx",
  "lengthMeters": 120,
  "floorPlanUrls": ["https://..."],
  "exampleVideoUrl": "https://...",
  "ruleNote": "...",
  "creatorId": "uuid",
  "recordCount": 42,
  "leaderboardSummary": {
    "topRecord": {
      "userId": "uuid",
      "nickName": "四驱小子",
      "lapTimeDisplay": "0:32.580"
    },
    "participantCount": 42
  },
  "createdAt": "...",
  "updatedAt": "..."
}
```

**副作用（登录用户）**：异步 `touchRecentVisit(userId, trackId)`，不阻塞响应。

**导航**：客户端按钮调用

```javascript
wx.openLocation({
  latitude: detail.location.lat,
  longitude: detail.location.lng,
  name: detail.name,
  address: detail.location.address
});
```

### 3.4 赛道列表

**GET `/tracks`**

| Query | 说明 |
|-------|------|
| page, pageSize | 分页 |
| keyword | 名称模糊搜索 |
| lat, lng | 用户坐标，传入则按距离升序 |
| sort | `distance`（需坐标）\| `latest`（默认） |

**列表项 `TrackListItem`**

```json
{
  "id": "uuid",
  "name": "...",
  "address": "摘要地址",
  "distanceMeters": 1250,
  "participantCount": 42,
  "topRecord": {
    "nickName": "四驱小子",
    "lapTimeDisplay": "0:32.580"
  }
}
```

**距离计算**（应用层 Haversine，单位米）：

```typescript
function haversineMeters(lat1, lng1, lat2, lng2): number;
```

未传坐标时 `distanceMeters = null`，排序用 `created_at DESC`。

### 3.5 我管理的赛道

**GET `/tracks/mine`**

- 需登录
- `WHERE creator_id = auth.userId`
- 卡片额外字段：`recordCount`（即 `tracks.record_count`）

**主理人工作台 UI 数据**

| 展示项 | 来源 |
|--------|------|
| 名称 | tracks.name |
| 位置摘要 | tracks.address 截断 |
| 入榜人数 | tracks.record_count |
| 待处理事项 | v1 固定 0（预留字段） |

### 3.6 最近访问

**GET `/tracks/recent`**

- 需登录
- 从 `recent_track_visits` 按 `visited_at DESC` 取 3 条
- JOIN tracks 返回 `TrackListItem` 简化版

**touchRecentVisit 逻辑**

```
INSERT ... ON DUPLICATE KEY UPDATE visited_at = NOW()
DELETE overflow WHERE user 保留最新 3 条（或查询 LIMIT 3 不物理删）
```

MVP：查询 `LIMIT 3` 即可，可不物理删除旧记录。
