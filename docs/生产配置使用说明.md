# 生产配置使用说明

| 版本 | 日期 | 说明 |
|------|------|------|
| v1.0 | 2026-06-25 | 初版，对应生产配置模板 |
| v1.1 | 2026-06-25 | 补充无 CDN（COS 直链）方案 |

> 模板文件：
> - 服务端：`server/.env.production.example`
> - 小程序：`miniapp/config.production.example.ts`
>
> 开发环境配置见 `server/.env.example` 与 `miniapp/config.ts`。

---

## 1. 使用前准备

上线前需先具备以下云资源（规格参考 [架构设计](./tech/架构设计.md) §8）：

| 资源 | 用途 |
|------|------|
| 云服务器 + HTTPS 域名 | API 服务（Nginx 反代 + Node） |
| MySQL 8 | 业务数据库 |
| Redis 7 | 榜单缓存、发帖/评论/上传限流（建议启用） |
| 腾讯云 COS | 图片/视频存储（CDN 可选，见 §4） |
| 腾讯位置服务 Key | 地图选点、逆地理编码 |
| 微信小程序 | AppID / AppSecret，并完成域名白名单配置 |

## 2. 服务端配置

### 2.1 创建生产 `.env`

在项目根目录执行：

```bash
cp server/.env.production.example server/.env
```

编辑 `server/.env`，将所有占位符替换为真实值。**`.env` 已在 `.gitignore` 中，切勿提交到 git。**

### 2.2 变量说明

| 变量 | 必填 | 说明 |
|------|:----:|------|
| `NODE_ENV` | ✓ | 固定为 `production` |
| `PORT` | ✓ | 进程监听端口，默认 `3000`（通常由 Nginx 反代） |
| `DATABASE_URL` | ✓ | MySQL 连接串，格式 `mysql://用户:密码@主机:端口/库名` |
| `REDIS_URL` | 建议 | Redis 连接串；留空则限流与榜单缓存降级为直查 DB |
| `JWT_SECRET` | ✓ | ≥32 字符随机串，**必须与开发环境不同** |
| `JWT_EXPIRES_IN` | ✓ | Token 有效期（秒），默认 `604800`（7 天） |
| `WECHAT_APP_ID` | ✓ | 小程序 AppID |
| `WECHAT_APP_SECRET` | ✓ | 小程序 AppSecret |
| `WECHAT_MOCK` | ✓ | 生产固定 `false` |
| `WECHAT_REALNAME_MOCK` | ✓ | 生产固定 `false` |
| `ADMIN_OPEN_IDS` | ✓ | 管理员 openId，逗号分隔；登录后自动获得 admin 角色 |
| `COS_SECRET_ID` | ✓ | 腾讯云 API 密钥 SecretId |
| `COS_SECRET_KEY` | ✓ | 腾讯云 API 密钥 SecretKey |
| `COS_BUCKET` | ✓ | COS 存储桶名称 |
| `COS_REGION` | ✓ | 桶所在地域，如 `ap-guangzhou` |
| `CDN_BASE_URL` | ✓ | 媒体对外 URL 前缀（无末尾 `/`）。可用 CDN 域名，或 COS 默认域名（无 CDN 方案，见 §4） |
| `MEDIA_ENV_PREFIX` | ✓ | 对象路径前缀，生产建议 `prod` |
| `MEDIA_MOCK` | ✓ | 生产固定 `false` |
| `TENCENT_MAP_KEY` | ✓ | 腾讯位置服务 Key（服务端逆地理编码） |
| `TENCENT_MAP_SK` | ✓ | 位置服务签名校验 SK |

### 2.3 生产环境禁止设置的变量

以下变量仅用于本地开发，**生产 `.env` 中不要出现**：

| 变量 | 原因 |
|------|------|
| `MOCK_LOGIN_CODE` | 会绕过真实微信登录 |
| `MOCK_MEDIA_HOST` | 会使用本地 mock 媒体服务 |

### 2.4 生成 JWT 密钥示例

```bash
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
```

### 2.5 获取管理员 openId

1. 生产环境完成微信登录配置后，用目标账号登录小程序一次。
2. 在服务端日志中查看 `[auth] login success ... openId=...`，或从数据库 `users` 表读取。
3. 将 openId 填入 `ADMIN_OPEN_IDS`。

## 3. 小程序配置

### 3.1 应用生产配置

将 `miniapp/config.production.example.ts` 的内容复制到 `miniapp/config.ts`，替换占位符：

| 变量 | 生产值 |
|------|--------|
| `API_BASE` | `https://你的API域名/api/v1`（须为 HTTPS） |
| `USE_MOCK_FALLBACK` | `false` |
| `MOCK_LOGIN_CODE` | `''`（空字符串） |
| `MOCK_REALNAME_VERIFY` | `false` |
| `TENCENT_MAP_SUBKEY` | 在 lbs.qq.com 申请并绑定 AppID 的 subkey |

### 3.2 编译 TypeScript

```bash
npm run compile:miniapp
```

此命令会将 `config.ts` 编译为 `config.js`，微信开发者工具加载的是 `.js` 文件。

### 3.3 开发者工具设置

提审前在 `miniapp/project.config.json` 中：

- 将 `"urlCheck": false` 改为 `true`（或保持 false 仅用于联调，提审前务必改回 true）
- 确认 `"appid"` 与生产小程序一致

## 4. 媒体存储：CDN 与无 CDN 两种方案

本项目上传走 COS 预签名 PUT；展示时使用 `CDN_BASE_URL` 拼出的 `publicUrl`（`<image>` / `<video>` 直接加载）。**MVP 可不接入 CDN**，改用 COS 默认域名直链。

### 4.1 方案对比

| | 方案 A：COS + CDN | 方案 B：COS 直链（无 CDN） |
|--|-------------------|---------------------------|
| 桶访问权限 | **私有读写** | **公有读私有写** |
| `CDN_BASE_URL` | `https://cdn.example.com` | `https://{bucket}.cos.{region}.myqcloud.com` |
| 上传 | 预签名 PUT → COS | 相同 |
| 播放/预览 | CDN HTTPS 回源 | COS 默认 HTTPS 域名 |
| 成本 | CDN 流量费 | 仅 COS 流量费 |
| 安全 | 桶私有，可配 Referer 防盗链 | 知道完整 URL 即可访问（路径含 UUID，难猜测） |
| 适用 | 正式上线、视频较多 | MVP、用户量小、快速上线 |

> **重要**：若桶为「私有读写」且未接 CDN，小程序内图片/视频会 **403 无法展示**。当前代码未实现读取时的预签名 GET，因此无 CDN 时必须改为公有读私有写。

### 4.2 方案 A：COS + CDN（推荐正式上线）

1. COS 桶设为 **私有读写**。
2. CDN 控制台添加加速域名，源站指向该桶，开启 HTTPS。
3. `.env` 配置：

```env
COS_BUCKET="mini-4wd-1333410420"
COS_REGION="ap-beijing"
CDN_BASE_URL="https://cdn.example.com"
MEDIA_MOCK=false
```

### 4.3 方案 B：无 CDN（COS 直链）

#### 步骤 1：创建/调整 COS 桶

| 配置项 | 值 |
|--------|-----|
| 访问权限 | **公有读私有写**（上传仍走预签名，不受影响） |
| 跨域 CORS | 允许 `PUT`、`GET`、`HEAD`（见下方） |

CORS 示例（桶设置 → 跨域访问）：

| 项 | 值 |
|----|-----|
| Origin | `*` |
| Methods | `PUT`, `GET`, `HEAD` |
| Allow-Headers | `Content-Type`, `Content-Length`, `x-cos-*` |

#### 步骤 2：配置 `.env`

`CDN_BASE_URL` 填 COS 控制台显示的 **默认 HTTPS 域名**（非 CDN）：

```env
COS_SECRET_ID="AKIDxxxxxxxx"
COS_SECRET_KEY="xxxxxxxx"
COS_BUCKET="mini-4wd-1333410420"
COS_REGION="ap-beijing"
CDN_BASE_URL="https://mini-4wd-1333410420.cos.ap-beijing.myqcloud.com"
MEDIA_ENV_PREFIX=prod
MEDIA_MOCK=false
```

上传成功后 `publicUrl` 示例：

```
https://mini-4wd-1333410420.cos.ap-beijing.myqcloud.com/prod/{userId}/record/video/xxx.mp4
```

#### 步骤 3：微信公众平台域名（无 CDN）

| 域名类型 | 填什么 |
|----------|--------|
| request 合法域名 | API 域名 + COS 默认域名（**上传**走 `wx.request` PUT 到预签名 URL） |
| downloadFile 合法域名 | COS 默认域名（**播放/预览** `<image>` / `<video>`） |

示例（桶在北京）：

| 类型 | 域名 |
|------|------|
| request | `https://api.example.com` |
| request | `https://mini-4wd-1333410420.cos.ap-beijing.myqcloud.com` |
| downloadFile | `https://mini-4wd-1333410420.cos.ap-beijing.myqcloud.com` |

> 本项目 COS 上传使用 `wx.request` + PUT，**不是** `wx.uploadFile`，因此 COS 域名主要加在 **request** 与 **downloadFile**，无需单独配置 uploadFile（除非后续改实现）。

#### 步骤 4：验证

- [ ] 上传后 `publicUrl` 以 `https://...cos.ap-beijing.myqcloud.com/prod/` 开头
- [ ] 浏览器或 curl 访问该 URL 可返回 200（非 403）
- [ ] 小程序赛道页/成绩页可正常显示图片与视频

## 5. 微信公众平台域名配置

在 [微信公众平台](https://mp.weixin.qq.com) → 开发 → 开发管理 → 开发设置 → 服务器域名：

**方案 A（CDN）**

| 域名类型 | 填什么 | 示例 |
|----------|--------|------|
| request 合法域名 | API + COS 域名 | `https://api.example.com`、`https://mini-4wd-1333410420.cos.ap-beijing.myqcloud.com` |
| downloadFile 合法域名 | CDN 域名 | `https://cdn.example.com` |

**方案 B（无 CDN）** — 见 §4.3 步骤 3。

**通用注意：**

- 所有域名必须支持 HTTPS，且证书有效。
- `API_BASE` 填的是带路径的完整 API 前缀；域名白名单只填主机名部分。
- 地图组件 subkey（`TENCENT_MAP_SUBKEY`）需在 [腾讯位置服务控制台](https://lbs.qq.com) 绑定小程序 AppID。
- 主理人实名校验依赖跳转「微信城市服务」小程序，需在微信侧完成城市服务相关开通与联调。

## 6. 配置完成后部署流程

```bash
# 1. 安装依赖
npm install

# 2. 编译服务端
npm run build:server

# 3. 执行数据库迁移（首次部署）
npm run db:migrate

# 4. 启动服务（生产机示例）
npm run start -w server
# 或使用 PM2：pm2 start server/dist/index.js --name mini4wd-api

# 5. 编译小程序
npm run compile:miniapp

# 6. 微信开发者工具 → 上传 → 提交审核
```

生产环境**不要**执行 `db:seed`（种子数据仅用于开发）。

## 7. 上线验证清单

配置与部署完成后，逐项验证：

### 服务端

- [ ] `GET https://api.example.com/api/v1/health` 返回 `{ status: "ok" }`
- [ ] `redis` 字段为 `true`（若已配置 Redis）
- [ ] 数据库连接正常，迁移已全部执行

### 登录与权限

- [ ] 小程序可完成真实微信登录（非 mock openId）
- [ ] 管理员账号登录后「我的」页显示「管理员」，可进入管理后台

### 媒体

- [ ] 上传赛道平面图 / 圈速视频成功
- [ ] 返回的 `publicUrl` 为 HTTPS，浏览器/小程序可访问（CDN 或 COS 直链均可）
- [ ] 确认 `MEDIA_MOCK=false` 且未配置 `MOCK_MEDIA_HOST`

### 地图与主理人

- [ ] 创建赛道时地图选点、逆地理编码正常
- [ ] 主理人申请可跳转城市服务完成实名校验（非 mock 流程）

### 小程序

- [ ] `USE_MOCK_FALLBACK=false`，断网/API 失败时不展示假数据
- [ ] `project.config.json` 中 `urlCheck: true` 下所有接口正常

| 配置项 | 开发（`.env.example`） | 生产（`.env.production.example`） |
|--------|------------------------|-----------------------------------|
| `NODE_ENV` | `development` | `production` |
| `WECHAT_MOCK` | `true` | `false` |
| `WECHAT_REALNAME_MOCK` | `true` | `false` |

## 9. 常见问题

### 图片/视频上传成功但无法显示（403）

- 无 CDN 方案：确认桶为 **公有读私有写**，不是私有读写。
- 确认 `CDN_BASE_URL` 与桶默认域名一致（含 `-APPID` 后缀与正确 `region`）。
- 微信公众平台 **downloadFile** 合法域名已添加 COS 域名。

### 登录报「登录凭证无效」

- 检查 `WECHAT_APP_ID` / `WECHAT_APP_SECRET` 是否与当前小程序一致。
- 确认 `WECHAT_MOCK=false`，且小程序 `appid` 匹配。

### 上传媒体失败或 URL 为 `http://127.0.0.1/mock-media/...`

- COS 四项（`COS_SECRET_ID/KEY/BUCKET/REGION`）未配全，或 `MEDIA_MOCK` 未设为 `false`。
- 检查 `CDN_BASE_URL` 是否为 HTTPS。

### 小程序 request 失败 / 域名不在白名单

- 确认 `API_BASE` 域名已加入微信公众平台 request 合法域名。
- 域名必须 HTTPS；自签证书不被微信接受。

### 地图不显示 / 选点失败

- 客户端 `TENCENT_MAP_SUBKEY` 与服务端 `TENCENT_MAP_KEY` 应为同一 Key 体系下的配置。
- subkey 须绑定小程序 AppID；服务端 SK 用于签名接口。

### 管理员入口不显示

- 确认目标用户 openId 已写入 `ADMIN_OPEN_IDS`（逗号分隔，无空格或 trim 后有效）。
- 修改后需该用户**重新登录**以刷新角色。
